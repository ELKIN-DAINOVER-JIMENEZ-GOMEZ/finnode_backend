🏗️  ARQUITECTURA IMPLEMENTADA:
═════════════════════════════════════════════════════════════════════════════
SERVICIOS:
──────────────────────────────────────────────────────────────────────────
  1. API Gateway (8080)           - Puerta de entrada
  2. Auth Service (8081)          - Autenticación y autorización
  3. Account Service (8082)       - Gestión de cuentas
  4. Payment Orchestrator (8083)  - Orquestación de pagos
  5. Ledger Service (8084)        - Registros de transacciones
INFRAESTRUCTURA:
──────────────────────────────────────────────────────────────────────────
  - PostgreSQL 16 Alpine          - Base de datos (puerto 5532)
  - Apache Kafka 3.7              - Message broker (puerto 9092)
  - Docker Network: finnode-network (tipo: bridge)
  - Volumen: postgres-data (persistencia de BD)

🚀 PRIMEROS PASOS:
═════════════════════════════════════════════════════════════════════════════
1. Navegar al directorio:
   $ cd /home/Dainover/Documentos/finnode
2. Crear archivo .env (opcional pero recomendado):
   $ cp .env.example .env
3. Iniciar todos los servicios (elige uno):
   Opción A - Usar docker-compose directamente:
   $ docker-compose up -d
   Opción B - Usar script bash:
   $ ./finnode-docker.sh start
   Opción C - Usar make:
   $ make start
4. Verificar que todo está corriendo:
   Opción A:
   $ docker-compose ps
   Opción B:
   $ ./finnode-docker.sh health
   Opción C:
   $ make health
5. Acceder a los servicios:
   API Gateway:              http://localhost:8080
   Auth Service:             http://localhost:8081
   Account Service:          http://localhost:8082
   Payment Orchestrator:     http://localhost:8083
   Ledger Service:           http://localhost:8084
   PostgreSQL:               localhost:5532
   Kafka:                    localhost:9092

🔧 COMANDOS ÚTILES RÁPIDOS:
═════════════════════════════════════════════════════════════════════════════
Ver estado:
  docker-compose ps
  ./finnode-docker.sh status
  make status
Ver logs en tiempo real:
  docker-compose logs -f
  ./finnode-docker.sh logs
  make logs
Ver logs de un servicio:
  docker-compose logs -f auth-service
  ./finnode-docker.sh logs auth-service
  make logs-auth
Entrar a shell de contenedor:
  docker-compose exec auth-service /bin/sh
  ./finnode-docker.sh shell auth-service
  make shell-auth
Entrar a PostgreSQL:
  docker-compose exec postgres psql -U postgres
  ./finnode-docker.sh db-shell
  make db-shell
Detener servicios:
  docker-compose down
  ./finnode-docker.sh stop
  make stop
Reiniciar servicios:
  docker-compose restart
  ./finnode-docker.sh restart
  make restart
⚠️  NOTAS IMPORTANTES:
═════════════════════════════════════════════════════════════════════════════
• Esta configuración es SOLO PARA DESARROLLO
• Las contraseñas están hardcodeadas (cambiar en producción)
• Los puertos 5532, 8080-8084, 9092 deben estar disponibles
• Requiere ~3GB de espacio en disco
• Requiere 8GB+ de RAM (se recomienda 12GB+)
• Los datos de PostgreSQL persisten en el volumen "postgres-data"
• Para borrar todo incluyendo datos: docker-compose down -v
